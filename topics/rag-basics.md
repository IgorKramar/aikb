# Что такое RAG (поиск с генерацией)

**RAG** (Retrieval-Augmented Generation) — это техника, которая позволяет ИИ использовать актуальную информацию из внешних источников при генерации ответов.

## Проблема, которую решает RAG

Обычные модели обучаются на данных до определённого момента и не знают:
- Свежих новостей
- Информации из твоих документов
- Данных из конкретной базы знаний

RAG решает это, добавляя к промпту релевантную информацию из внешних источников.

## Как работает RAG?

1. **Вопрос пользователя**: «Какая погода в Москве сегодня?»
2. **Поиск информации**: система ищет актуальную информацию о погоде в Москве
3. **Добавление к контексту**: найденная информация добавляется к промпту
4. **Генерация ответа**: модель использует эту информацию для ответа

## Простой пример

Без RAG:
- Вопрос: «Что в меню ресторана "Вкусно и точка" на сегодня?»
- Ответ модели: «Я не знаю актуального меню, так как мои знания ограничены...»

С RAG:
- Система находит актуальное меню на сайте ресторана
- Добавляет его к промпту
- Модель генерирует ответ на основе реального меню

## Где используется RAG?

- **Чат-боты с базой знаний**: отвечают на вопросы на основе внутренних документов компании
- **Поисковые системы с ИИ**: Perplexity, Bing Chat используют RAG для актуальных ответов
- **Помощники для документов**: могут отвечать на вопросы по содержимому PDF, Word и других файлов
- **Корпоративные ассистенты**: используют внутренние базы данных и документы

## Технические детали

RAG обычно включает:
1. **Векторную базу данных**: для быстрого поиска релевантной информации
2. **Эмбеддинги**: для преобразования текста в векторы для поиска
3. **Поисковую систему**: для нахождения наиболее релевантных документов
4. **Языковую модель**: для генерации ответа на основе найденной информации

## Преимущества RAG

✅ Модель может использовать актуальную информацию
✅ Можно работать с большими объёмами данных
✅ Ответы основаны на реальных источниках
✅ Легче контролировать, откуда взялась информация
