# Температура и другие параметры модели

При работе с API моделей (не в обычном чате, а при программировании) можно настраивать параметры, которые влияют на то, как модель генерирует ответы.

## Температура (Temperature)

**Температура** — это параметр, который контролирует «креативность» или «случайность» ответов модели.

- **Низкая температура (0.1-0.3)**: модель даёт более предсказуемые, точные ответы. Хорошо для фактов, технических задач, переводов
- **Средняя температура (0.7-0.9)**: баланс между точностью и креативностью. Хорошо для большинства задач
- **Высокая температура (1.0-2.0)**: модель более креативна, но может быть менее точной. Хорошо для творческих задач, генерации идей

### Примеры:

Температура 0.2 (низкая):
- Вопрос: «Столица Франции?»
- Ответ: «Париж» (всегда одинаково)

Температура 1.5 (высокая):
- Вопрос: «Напиши стихотворение про кота»
- Ответ: каждый раз разное, более креативное стихотворение

## Top-p (Nucleus Sampling)

**Top-p** — параметр, который ограничивает, из каких слов модель может выбирать следующее слово.

- **Низкий top-p (0.1-0.5)**: модель выбирает только из самых вероятных слов — более предсказуемо
- **Высокий top-p (0.9-1.0)**: модель может выбирать из большего набора слов — более разнообразно

Обычно используется вместе с температурой.

## Max tokens

**Max tokens** — максимальное количество токенов в ответе модели. Ограничивает длину ответа.

- Если установить max_tokens=100, модель остановится после ~100 токенов
- Полезно, когда нужен короткий ответ

## Frequency penalty и Presence penalty

Эти параметры помогают избежать повторений:

- **Frequency penalty**: наказывает за частое повторение одних и тех же слов
- **Presence penalty**: поощряет использование новых тем и слов

Полезно для длинных текстов, чтобы избежать зацикливания.

## Практические рекомендации

- **Для фактов и точных ответов**: temperature=0.2, top_p=0.1
- **Для обычных задач**: temperature=0.7, top_p=0.9 (значения по умолчанию)
- **Для творческих задач**: temperature=1.2, top_p=0.95
- **Для кода**: temperature=0.1-0.3 (нужна точность)
